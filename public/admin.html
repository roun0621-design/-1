양식 다운로드 (.xlsx)

1단계: 연맹 명단 업로드

연맹 엑셀 파일을 업로드하면 선수 + 종목 + 출전배정이 한번에 자동 등록됩니다.

엑셀 양식 (열 순서 고정):
A열: 팀명 * | B열: 선수명 * | C열: 성별(남/여) * | D열: 생년월일 | E열: 종목1 | F열: 종목2
G열~K열: 릴레이(4x100mR, 4x400mR, Mixed 등) — 참가 시 O 입력
L열: 바코드 — 사용자가 직접 등록한 바코드 번호
* 배번(BIB)은 이 단계에서 등록하지 않음 (조편성에서 배번 배정)
* 바코드는 시스템이 자동 생성하지 않음 (사용자 직접 입력)
* 1행은 헤더(무시됨), 2행부터 데이터. 종목 열 비워두면 선수만 등록됩니다.

2단계: 조편성 업로드

조편성 엑셀 파일을 업로드하면 해당 종목의 조/레인/배번이 배정됩니다.
파일에 포함된 종목만 변경되며, 포함되지 않은 종목은 영향 없습니다.

엑셀 양식:
A열: 성별(남/여/혼성) | B열: 종목 | C열: 라운드(예선/결승)
D열: 조 | E열: 그룹(A/B) — 5000m/10000m만 해당
F열: 순서(레인) | G열: 배번 | H열: 성명 | I열: 소속
릴레이: 배번 비우고 성명에 팀명 입력 | 배번 비워두면 NULL 저장 (자동 배정 없음)

선수 명단 엑셀 업로드 (별도)

종목 배정 없이 선수만 등록합니다. 종목 배정은 소집실 → 선수 추가 또는 관리탭에서 별도 진행하세요.

엑셀 양식 (열 순서 고정):
A열: 배번(BIB) — 비워둘 수 있음 | B열: 선수명 * | C열: 소속 | D열: 성별(남/여) * | E열: 바코드(선택)
* 배번/바코드를 비워두면 NULL 저장 (자동 생성하지 않음)
* 1행은 헤더(무시됨), 2행부터 데이터

3단계: BIB 번호 일괄 수정

엑셀 파일에서 배번을 읽어 기존 선수와 이름+소속+성별로 매칭하여 BIB만 업데이트합니다.
기존 기록(결과, 조편성 등)에 영향 없음 — BIB은 표시용 필드입니다.

엑셀 양식 (열 순서 고정):
A열: 배번(BIB) * — 새로 부여할 배번 | B열: 선수명 * | C열: 소속 * | D열: 성별(남/여) 선택
* 매칭 키: 선수명 + 소속 (성별 열이 있으면 + 성별도 비교)
* 1행은 헤더(무시됨), 2행부터 데이터 | 성별 열 없어도 이름+소속으로 매칭 가능

종목 리스트 엑셀 업로드

종목만 일괄 등록합니다. (혼성경기는 세부 종목 자동 생성)

엑셀 양식 (열 순서 고정):
A열: 종목명 * | B열: 카테고리 | C열: 성별 | D열: 라운드
카테고리: track, 트랙, field_distance, 필드(거리), field_height, 필드(높이), relay, 릴레이, combined, 혼성, road, 도로
성별: 남/M, 여/F, 혼성/X  |  라운드: 결승/final(기본), 예선/preliminary, 준결승/semifinal
* 1행은 헤더(무시됨), 2행부터 데이터. 카테고리/성별/라운드 미입력 시 기본값: track/남/결승

팝업 편집

🏆 기록 관리 시스템 운영 가이드 NR / DR / CR · 자동 감지 워크플로

신기록 자동 감지 시스템의 동작 원리와 운영 규칙입니다. 관리자/운영자가 숙지해야 할 내용입니다.

기록 종류
  • NR (National Record) — 한국기록
  • DR (Division Record) — 부별기록 (13개 부: 초/중/고/대/일반/공개 × 남녀 + 혼성)
  • CR (Competition Record) — 대회기록 (시리즈별, 예: KTFL/광주올림픽시리즈 등)
자동 감지 흐름
  1. 경기 결과 입력 시 NR/DR/CR 동시 비교
  2. 갱신 후보를 승인 대기 큐로 보냄
  3. 관리자가 검토 후 승인 / 거부
  4. 승인 시 자동으로 공식 기록 갱신 + 공개 페이지 배너 표시
자동 제외 규정
  • 풍속 +2.0 m/s 초과 — 100m / 200m / 100mH / 110mH / 멀리뛰기 / 세단뛰기에서 자동 참고기록 처리 (신기록 불인정, 토스트 안내)
  • DQ / DNS / DNF / NM — 비유효 상태는 비교 대상에서 제외
  • 혼성종목(combined) — 자체 점수체계로 별도 관리
  • 예선/준결승 — 결승 race만 비교 대상
운영 권장 사항
  • 대회 생성 시 시리즈 연결해야 CR 자동 감지 작동
  • 승인 큐는 기록 관리 탭 상단에 자동 표시 (pending 0건이면 숨김)
  • 풍속 측정값은 m/s 단위, 소수 1자리로 입력
  • 의심 케이스는 거부 후 사유 메모로 기록 (감사 추적)
